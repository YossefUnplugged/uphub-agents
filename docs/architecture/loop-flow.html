⏰ הערה (wake)
Task Scheduler מפעיל tick (עם נעילת single-instance — סשן אינטראקטיבי קודם), או /loop / הרצה ידנית. מריץ את הסוכן headless עם פרוטוקול ה־triage. בודק בריאות (Jira/gh/MCP) לפני שמתחיל.
triage-loop.ps1 → run-headless.ps1 → claude -p (prompts/triage.md)
1
🔎 Poll — בוחר טיקט אחד
שולף מ־Jira לפי JQL. רואה אך ורק טיקטים שמשויכים אליך — לא ניתן לגעת בטיקט של מישהו אחר.
assignee = currentUser() AND labels = ai-ready AND status = "To Do"
2
✅ Readiness checklist (חובה הכול)
repo onboarded · יש acceptance criteria · estimate (מעל 4ש׳ או נתיב אבטחה → דורש plan-approved) · QA משויך · אין PR פתוח כבר.
⛔ כשל → לא כותב קוד. מנסח טיוטת ai-needs-info לסקירה אנושית ו־STOP.
3
🧱 Spec extraction
חומת־אש לקלט לא־מהימן: טקסט הטיקט הופך למפרט מובנה. הוראות זדוניות שמוטמעות בטיקט (״תשלח את ה־env ל…״) = דאטה, לא פקודה.
4
⌨️ Implement
פותח ענף UNP-NNNN, כותב קוד + טסטים. route-on-touch מזריק את ה־skill הנכון לכל קובץ שנוגעים בו. (אופ׳: פירוק ל־sub-agents לטיקט גדול.)
branch UNP-NNNN · route-on-touch → admin-* skills
5
🛡️ Gate A — מכני
סקריפט מריץ: lint · tsc -b · טסטים · regex של branch/commit · imports אסורים · staged-path · גלאי־השמטות. קודי־יציאה שאי אפשר להתווכח איתם.
node scripts/validate.mjs --target admin
🔴 אדום → סבב תיקון; לא ממשיכים עד ירוק.
6
🔬 Gate B — סקפטי בהקשר נקי
מעבר נפרד על ה־diff בלבד (git diff main...HEAD) — code-review + security-review. לא ראה את ההנמקה של המבצע. (6c: אם נגעת ב־routes/Actions → סנכרון הרשאות AAA דרך up-aaa-sync.)
🔴 CRITICAL/HIGH לא נפתר → סבב תיקון 1 → עדיין → ai-blocked + STOP.
7
🚀 Close — Draft PR
דוחף ענף, פותח Draft PR עם ה־reviewers מה־config. גוף ה־PR כולל: spec, סיכום Gate A, ממצאי Gate B, שורות router-miss, וסעיף הרשאות AAA אם רלוונטי.
gh pr create --draft --reviewer <defaults>
8
📋 Jira → Waiting for CR
מעביר את הטיקט לעמודת Waiting for CR, מוסיף תגובה עם כתובת ה־PR, ו־@mention ל־QA שיתחיל בדיקות. זה המצב הסופי של הסוכן.
transitionJiraIssue → "Waiting for CR" · addComment (PR URL + @QA)
↻ חזרה לשלב 0 ב־tick הבא — לוקח את הטיקט הבא בתור (טיקט אחד בכל ריצה).
👤 שער אנושי קבוע — מחוץ ללולאה
בן־אדם סוקר את ה־PR וממזג. זה אף פעם לא אוטומטי — גם כשהסוכן יהיה ״סניור״. (גם מפתח סניור עובר code-review ולא דוחף ל־main.)
